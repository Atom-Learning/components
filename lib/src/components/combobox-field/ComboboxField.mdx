`ComboboxField` wraps Reach UI's `Combobox` component with default styling and accepts all the same props. See [the Reach UI docs](https://reach.tech/combobox/) for details.

Arbitrary components can be nested inside the `ComboboxField.Popover` to allow for more advanced usecases, e.g. adding a new value to the list of suggestions

Note that when combined with a `Label`, the `id` that matches the `Label`'s `htmlFor` prop should go on the `ComboboxField.Input`.

```jsx live
<Form onSubmit={() => undefined}>
  <Box css={{ width: '400px' }}>
    <Label css={{ mb: '$3' }} htmlFor="someid">
      What's your favourite fruit?
    </Label>
    <ComboboxField onSelect={console.log} openOnFocus>
      <ComboboxField.Input name="new-input" label="new input" validation={{
        min: {
          value: 3,
          message: 'You must have at least 3 cats!'
        }
        }} />
      <ComboboxField.Popover>
        <ComboboxField.List>
          <ComboboxField.Option value="Apple" />
          <ComboboxField.Option value="Banana" />
          <ComboboxField.Option value="Cranberry" />
          <ComboboxField.Option value="Dragon fruit" />

          <Flex css={{ alignItems: 'center', p: '$2' }}>
            <Input size="sm" placeholder="New fruit" />
            <Button
              size="sm"
              css={{ ml: '$2' }}
              onClick={() =>
                alert('Nest other interactive UI here for advanced usecases')
              }
            >
              Add a new fruit
            </Button>
          </Flex>
        </ComboboxField.List>
      </ComboboxField.Popover>
    </ComboboxField>
  </Box>
</Form>
```
